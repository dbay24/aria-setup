Basic ARIA Definitions

  • Roles: Roles describe widgets that aren't otherwise available in HTML 4, such as sliders and tabs. The role of an element should not change. Instead, remove the original element and replace it with an element with the new role
  • Properties: Properties describe characteristics of these widgets, such as if they are draggable, have a required element, or have a popup associated with them. `Property` values may change but are less likely to change than state values, this is not a rule Examples of properties: aria-haspopup, aria-controls, aria-autocomplete
  • States: States describe the current interaction state of an element, informing the assistive technology if it is busy, disabled, selected, or hidden. `State` values change frequently based on user interaction Examples of states: aria-checked, aria-disabled, aria-hidden, aria-grabbed, aria-required
  • Owned Elements: An 'owned element' is any DOM descendant of the element, any element specified as a child via aria-owns, or any DOM descendant of the owned child.
